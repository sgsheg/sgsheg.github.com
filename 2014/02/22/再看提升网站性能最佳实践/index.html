再看提升网站性能最佳实践

  1. 第一类 [内容]

    • 尽量减少HTTP请求

      • 合并文件，如果可能，合并你的css,js文件

      • 使用css sprites合并你的图片

      • 在某些特殊页面可以考虑使用图片地图(map)，但是不推荐

      • 使用内联图像（data:URL scheme），目前多数浏览器尚未支持

    • 减少DNS查找

      • 一般情况下返回给定域名对应的IP地址会花费20到120毫秒的时间，而且在这个过程中都不会直到DNS查找完毕

      • 缓存DNS

      • 减少主机名的数量可以减少dns查找次数

    • 避免跳转

      • 这里的跳转指的是301或者302.html文件在被加载前（图像，flash等）都不会被下载，所以跳转会拖延页面中所有元素，减低用户体验。

      • 但是如果必须要跳转的话，最好方法是使用标准的3XXHTTP状态码，这主要确保“后退”按钮可以正确使用。

      • 避免url有/时被忽略，因为带有/的网址包含301跳转

    • 缓存Ajax

      • 提高ajax性能的措施中最重要的方法就是使响应具有可缓存性，一般可通过为文件头指定Expires或Cache-Control来优化。

    • 推迟加载

      • 仔细查看网页，那些内容是页面呈现时候所必须要加载的？那些结构可以稍后加载。

      • 把整个过程按照onload事件分隔为两部分，js是一个理想选择。

      • 那些你有用于拖放和动画的js,那么就等它稍后加载。

    • 预加载

      • 预加载是在浏览器空闲时候请求将来可能会用到的页面内容（如图像，样式和文本）

      • 当用户在访问下一个页面时候，页面大部分内容都已经加载完毕了。

      • 常用方法

        • 无条件加载

          • 触发onload事件，直接加载额外的页面内容。

        • 有条件加载

          • 根据用户的操作来有根据的进行加载

          • 有预期的加载：载入重新设计的页面进行预加载。

    • 减少DOM元素数量

      • 一个复杂页面意味着更多的下载数据，同时也意味着js遍历dom的效率越慢。

      • 先规划好页面，尽量优化dom

    • 用域名划分页面内容

      • 把页面内容分为若干部分，不同部分放在不同的域名下。

    • 将iframe数量降到最低

      • iframe优点：解决加载缓慢的第三方内容如图标和广告的加载问题

    • 避免404

      • http请求消耗是很大的，因此使用http请求获取一个没有用处的响应是完全没有必要的，只会降低用户体验而不会有一点好处

  2. 服务器

    • 使用内容分发网络CDN

      • 把你的网站内容分散到多个，位于不同地域的服务器上可以加快加载

    • 为文件头指定Expires和Cache-Control

      • 对于静态内容：设置文件头过期时间Expires的值为“Never expire”（永不过期）

      • 对于动态内容：使用恰当的Cache-control文件头来帮助浏览器进行有条件的请求

    • Gzip压缩文件内容

      • 目前最流星也是最有效的压缩方式

    • 配置ETag

    • 尽早刷新输出缓冲

    • 使用GET来完成AJAX请求

    • 避免空的图像来源

  3. css

    • 把样式置于顶部

    • 避免使用css表达式

    • 用@代替import

    • 避免使用滤镜

  4. js

    • 把脚本置于页面底部

    • 使用外部css和js

    • 精简css和js

    • 剔除重复脚本

    • 减少dom访问

    • 开发智能事件处理请求

  5. cookie

    • 减少cookie体积

    • 对于页面内容使用无cookie域名

  6. image

    • 优化图像

      • 推荐工具Smush.it和imagemagick

    • 优化css spirite

    • 不要在html中缩放图像

      • 不要在html中通过指定宽和长来缩小图片

    • favicon.ico要小，并且可缓存

      • 文件尽量的小，最好小于1k

      • imagemagic可以帮助你创建小巧的favion

  7. Mobile

    • 保持单个内容小于25k

      • iphone不能缓存大于25k文件

      • 精简文本十分必要

    • 打包组件成复合文本
