《代码的未来》阅读下

  1. 作为系统描述语言的C和C++,作为商务的语言而入日中天的Java,以及在web领域中十分热门的Ruby,perl,Python,PHP等脚本语言,都属于主流语言.这些语言在成长过程中都吸收了过去一些语言的优点,例如Java借鉴了C++,Smalltalk和Lisp的优点,而Ruby则是在吸收了过去语言的基础上而独自发展的语言.

  2. 仅仅将过程进行抽象化还远远不够.几乎所有的过程都需要进行一定的输入输出操作,而不是与数据无关.在数据抽象化的延长线上,就自然而然产生了面向对象编程的概念.所谓抽象,就是抽象化数据本身.

  3. 因此面向对象和数据抽象化之间仅仅隔了薄薄的一层纸.21世纪面向对象已经是常识了.

  1. 讲述软件开发的一本名著《人月神话》作者写到:"无论使用什么语言,生产一条基本语句所需要的工数几乎是一定的".也就是说,如果描述相同的算法,A语言需要1000行,B语言只需要10行,只要采用B语言生产效率就提高了100倍.

  2. 在我看来,编程语言的进化动机,不是工具和语言本身的简化,而是将通过这些工具和语言所得到的结果(解决方案)更简介地表达出来.

  3. 因此我预测100年后的编程语言的样子,我认为应该是下面三种情况:

    1. 变化不大
    2. 使用编程语言来编程这个行为本身不存在了.
    3. 发明了采用更加抽象化写法的编程语言.<更加强调what,而对于如何解决问题的How部分的细节,则不再需要人类去问.

  1. 闭包-编程语言的新潮流.我们来比较以下四种语言.Java,Javascript,Ruby和Go.

  2. 在这四门语言中,Java和Go是比较侧重静态一侧的语言,而Ruby和Javascript则是比较偏重于动态一侧的语言.

  3. 在Javascript中与服务器进行异步通信的API叫做XMLHttpRequest,因此从它所衍生的手法被称为Ajax.

  4. Ruby能拥有现在的人气,基本上都是Ruby on Rails所作出的贡献.目前,作为服务器编程语言,Ruby的人气可谓无可撼动.

  1. Go是一种新兴的语言,但出身名门,由著名的UNIX开发者罗勃.派克和肯.汤普逊开发的,因此收到了广泛关注.

  2. 面向对象的语言都具备被称为多态(Polymorphism)或者动态绑定的动态特性.即,根据存放在变量中的对象的实际性质,自动选择一种合适的处理方法.这样的功能可以说是面向对象编程的本质.

  3. 所谓动态运行模式,就是运行中程序能够识别自身,并对自身进行操作.对程序自身进行操作的元编程,也被成为元编程.(Metaprogramming).

  4. 在Ruby和Javascript中,元编程是十分自然的.

  5. 程序"对自身进行操作"也被称为"反射".同样的意思,在Ruby大多被成为元编程,而在Java大多数叫做反射.

  6. 在动态语言中,有一种叫做"鸭子类型"(Duck Typing)的风格被广泛使用.

  7. If it walks like a duck and quacks like a duck, it must be a duck!如果像鸭子一样走路,像鸭子一样呱呱叫,则它一定是一只鸭子.我们可以推导出一个规则,即如果某个对象的行为和鸭子一模一样,那无论它真正的实体是什么,我们都可以把看做是一只鸭子.也就说,不考虑某个对象到底是哪一个类的实例,而只关心其拥有怎样的行为.这就是鸭子类型.

  8. 写一个go的小例子.

    package main
import "fmt"
func.main() {
    printf("hello,世界\n");
}

cmd: go build test.go
cmd: test.exe

  9. Go的一些新的特性.和C语言比较,Go中if等结构中并不用括号来括起来,而相应地,程序体的部分则是用花括号来括起来.

  10. else省略括号歧义的问题.在Perl的语法中也不允许省略花括号.说句题外话,Ruby在控制结构的划分中,没有使用花括号,而是使用了end,理由也是一样的.使用end的语言中来避免悬空导致的歧义.另外,Go语言中是没有while语法的.

  11. google奉行"不作恶"(Don't be evil)的信条,因此很多人觉得Go应该改名.
