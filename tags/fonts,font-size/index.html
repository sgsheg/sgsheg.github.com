web字体问题

你可以在WEB上使用任意单位来设置字体的大小,但主要的方法无非三种:像素,em和百分比.

像素,派卡(pica),英寸

  1. 像素一直是人们喜欢使用的字体大小单位,原因很简单,无论浏览器如何设置字体大小,你都能对其进行精确的控制大小.如果你把页面的字体大小设置为18px,那么每个浏览器都会将字体准确地显示为18px.但这里就有问题了.要更改字体的大小,就需要一个一个去进行更改!当用户放大或者缩小页面的时候,浏览器一般会对页面内容有两种操作方式.

    a. 简简单单将所有东西都放大,所以当访问者放大页面的时候,包括文本在内的所有的元素都会变大.

    b. 浏览器只会调整文本的大小,页面其他元素则保持不变.

  2. 长期以来,b都是常见浏览器实现的方式,并且仍有一些浏览器还在使用这种方式实现缩放功能.注意: IE9以下版本的IE浏览器,字体被设置的浏览器,是不能调节页面字体大小的.

  3. 无法调整字体大小的问题也同样出现在很多较老的,在触摸屏没有出现之前就有的设备上.

  4. 用像素来指定字体大小依然不是一种对未来特别友好的方式.由于不同的设备有着不同的屏幕大小和像素密度,因此以像素为单位的字体在这设备上看起来不错,但是另外一台设备上也许要么太大,要么太小.

  5. 使用像素作为字体大小的单位,是与web的灵活性原则背道而驰的原则.

  6. 使用派卡(pisa)或英寸,它们是绝对单位,因此设定多大就多大,与祖先元素的字体大小无关.在需要调整页面元素大小时候,必须一个一个地修改font-size,很麻烦.

em

  1. 使用em作为字体大小的单位是一种更加流行,更具有灵活性的方法!1em等于默认的字体大小.假设内容字体为16px,那么

    1em=16px

    2em=32px

  2. em可以跨浏览器进行缩放,而且他们也是级联的.从简化维护上看,这样相对地指定元素的字体大小,意味着你只需要调整初始化时的基准,其余部分就会自动地进行调整,而且是按照比例调整的.但级联也会让事情变糟!但其变化时候,基准也会随着进行变化.

  3. 如果你跟某个元素设定了相对元素,则该元素的字体大小要相对于最近的"被设定的字体大小的"祖先元素决定.

  4. 要注意设置字体时候,必须事先规划好了.

  5. 今天设备的屏幕尺寸可谓千差万别,既有特大的显示屏,又有很小的手机屏幕.在这种形势下,更容易缩放的相对大小应该是首选.

百分比

  1. 以百分比为单位的字体也是可缩放的,而且也是级联的.

  2. 虽然,百分比和em没有太大区别,而且em逐渐成为了WEB中更受欢迎的字体度量单位,但其实没有什么技术的原因.

  3. 你可以通过在body元素上以百分比为单位指定基准字体大小来绕过这个问题.

    body {

     font-size: 100%;

    }

  4. 在大多数浏览器中,默认的字体大小都是16px.通过将body字体设置为100%,你就可以保证页面内容是可缩放的,而且不会有夸大的现象,之后你就可以使用em来设置其他元素的字体大小.

    body {

     font-size: 100%;
 font-size: 1em;

    }

rem

  1. rem "root em".rem和em的最大区别:rem的大小与html元素有关.

    body {

     font-size: 100%;    /* equates to ~16px*/

    } h1 {

     font-size: 1.5em;   /* 24px / 16px*/

    } span {

     font-size: 1rem;    /* 16px / 16px*/ /*span继承了html元素字体大小*/

    }

  2. 但是对于rem,移动浏览器有不支持的情况.桌面浏览器IE9+,chorme,IOS4.0+,Android2.1提供了相应的支持.但是Opera Mobile却不支持.

  3. 为了应对不支持的情况,可以额外设置一个以像素为单位的备用方案.

    span {

     font-size: 16px;    /*  那些不支持rem的浏览器将会使用第一条像素声明,并忽略rem声明*/
 font-size: 1rem;    /*  支持rem的浏览器将会使用rem声明 */

    }
