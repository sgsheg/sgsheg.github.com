从离开到进入创业公司

开始的第一篇

晚上,在卫生间看到了[小道君]的一篇微信文章,我们要从[员工的辞职信息里面看到别的信息].于是才有了此文!

我想这样开始,我是12年才开始真正意义上的接触互联网.大学时代学的很多,但都不精巧.大学毕业后,才知道一个事实,活着,严格意义上好好的过着自己想过的生活是有点那个什么了/

"孩子,你想多了! 做事情,你做好一件事情已经成功了!莫要贪多,贪多嚼不烂"!

12年,第一家公司.这家公司严格意义上并非一家真正的互联网公司.老板的企业文化是[我要你们像流水线工人一般,好好干],你猜我想到了什么!台湾首富[郭台铭]的富士康.毕竟我在那里呆过,不符合人道.玩计算机的,都知道”freedom””opensource”是追求,但是至少在这里我感觉不到编码的快乐.如果你还在这样的公司,我可以肯定告诉你,你还是换家公司吧!再高的工资都给不了你快感!中间,有幸当上了项目负责人,但项目负责人不是项目经理,这是一个不讨好的工作,当年被别人面试的时候,发誓自己有一天要把这口气要回来,是不是有点小马哥哥的气势,可是当我听到[熟练使用notepad++],我就开始忍不住骂街了,亲,你好歹也说个[熟练使用VIM或者是Emacs]啊!

于是,人生第一次觉得没意义,这种熟悉的快感是扯淡.于是,向恩师[李文凯]转发,师傅笑道,[哦.呵呵].
于是,人生中有了口头禅.可惜,这道口头禅伤了我最亲的人.此话后说吧!<隐私,懂不懂>

有一朋友在福建写CMS,好吧,严格意义上是三个人,配置是这样的,[两个技术员,一个做技术支持],当然技术支持严格意义上就是销售.哥们悄悄伸出橄榄枝!
[come on baby,来我这吧!我这叫做创业.做自己的CMS,服务于中小企业,做草根站点,服务于广大草根站长.]
好吧,正式开启了我的[福建->陕西->北京].

去福建,情况和你想的有点不一样,24小时,干的死,干的吐!当然,南方天气压抑,暗淡.但最重要的是人的承受能力,作为程序员,其实,你有必要在你最好的状态发挥你最大的力量.而不是流水线工人,流水式的工作,不会让工作效率提升.我们可以熬夜工作,前提是兴奋和状态.人的体能是要均衡的. 当然,中医讲,人身体是一个均衡的,比如男人的精液,好吧,还是说’神’吧,像水龙头一般,就那么多水,口拧打了,亲,小心你的后半辈子.

程序是快乐的,如果是以时间和精力来耗费,二个字,不值!

终于有一天自己病了,躺到床上,想起了李宗盛的<凡人歌>,[你我皆凡人,生在人世间.终日奔波苦,一日不得闲].连躺两天,看到电脑就恶心.我感觉到了事情的可怕,于是和朋友彻夜长叹后,离开福建,去了另一个哥们那里,脑袋里只有一个字[玩].彻底的远离计算机,看自己之后还是否要继续和计算机打教导.

一夜之后,顺利达到陕西.当然,哥们带我爬’连城山’,喝’汉江茶’,吃’核桃饼’,看’兵马佣’,看”黑客与画家”.这是我心最静,从离开大学就没有的感觉.一番游玩以后,朋友问我,要不要再找份工作,此刻犹豫了,放弃,还是继续??
可惜,我对着一包烟苦思冥想后,嗯,我手贱地打开了计算机,此奥,Ubuntu,好吧!我还是喜欢计算机的嘛!

于是,奋笔一挥,我要回北京.6月份,开始到达北京,然后进入了所在公司.
可笑的是,和谈论的时候,我的人生理想是”创业,当CEO”为自己埋下了隐患.老板没事损我的时候,开头就是,”亲,小CEO,….”!

好吧,这就是2013年目前的回忆.

我们还是谈谈技术方面吧!聊一聊,最近使用到技术话题吧!要总结历史,才能更健康成长.

CSS性能问题

CSS性能问题

与网页接触很久了,从最初的了解信息,到后来的developer.最近的两个月内PHP写的少了,less和js倒是写的不少了,虽然服务器上将less解析为css的脚本是我写的,无奈在前端开发中又岂止当初的浮动,盒子模型.less和sass的出现,让CSS的书写更加顺手.

CSS的渲染速度决定于哪里?这个因素很大.对于php来说,smarty渲染,网络速度,页面加载速度,有很多的因素.但不管如何,知道CSS的一些好写法,未尝不可.要不然,像CSS3这样的屠龙刀,你要用它劈柴,呵呵,你还是换个吧,这么重的玄铁,还挺费手!

看完以后,总结如下:

开头第一句话,

Put CSS in the document head!!!

  1. id选择器和class选择器上,多数人觉得id比class快,但事实上二者的区别不大.可以不做考虑

  2. CSS3的绚丽特性犹如小米,才是页面渲染的最大考验.animation和@font-face,请谨慎使用.对于可以不使用CSS3属性,比如text-shadow,那就尽量不使用.

  3. 对于ul > li >a[title="sgsheg"]这样的代码,最先读到的部分是a[title="sgsheg"]

  4. 选择器的效率,从高往低来讲,依次是id > class > tag > universal.

  5. 不需要在使用id选择器时,使用ul#sgsheg,因为id是唯一的.ID selectors are unique by definition.

  6. 对于tag标签ul等,也要与class分开写.

  7. 后代选择器是性能最差的,譬如ul > li > a {},这样的使用是很糟糕的.记住,浏览器的渲染是从右往左的.这样的代码,只会更加的累赘.Why not give the a an class to render?对于这样的,为什么不给它个class,.alink { color: red}

  8. 在浏览器渲染页面时候,如果出现错误,那么它就会跳过这段代码,继续进行下面的.

  9. 在书写CSS的时候,you need think more,是否可以使用继承,继承才是代码艺术.就像jQuery那样,write less, do more.否则,我只能说”呵呵,请接受治疗!”

  10. 停止使用@import,请使用link引入.请把Javascript位于CSS之后.

  11. 在手机上,最牛叉的配置也赶不上十年前的电脑,所以开发手机网页要谨记,耍酷要看平台.

当然,原文部分在这里:

  1. Moving inline style blocks and elements from the document body to the document head improves rendering performance.

  2. Use the tag instead of CSS @import

  3. CSS 3 properties are some of the worst offenders when it comes to page performance. Especially animations and @font-face. Use them wisely.

  4. Generally speaking, if a page is slow to resize it’s likely a layout property (float, fluid width, etc.) and if it’s slow to scroll it’s likely an aesthetic property (such as shadows and gradients) that’s slowing the page down.

  5. Mobile is a thing and even the best mobile phones aren’t as powerful as the best computers from a decade ago.

ubuntu使用技巧1

Ubuntu使用技巧1

一年多来,都在Ubuntu上进行开发.Ubuntu可以说linux发行版中比较好用的一派,虽然比不上[苏西]的绚丽,但是已经足够使用.
Ubuntu是Debian的一个分支,安装软件方便简单.

sudo apt-get install + softname

当然我就把我常常用到的给列出来.

Ubuntu快速卸载软件.

sudo apt-get purge

来自于如下:

apt-get --purge remove ... is the the same as apt-get purge ..... 
purge removes global configuration files only.It does note remove virtual machines,so it's safe to use.

Ubuntu完全卸载libreOffice

sudo apt-get purge libreoffice*
sudo apt-get clean

Ubuntu卸载kazam.

sudo apt-get purge kazam

Ubuntu卸载VirtualBox

sudo apt-get purge virtualbox-*

Ubuntu卸载gnome和gnome-classic

sudo apt-get purge gnome-shell-*

更多详见: 

http://askubuntu.com/questions/65200/remove-gnome-shell-completely-after-installing-it

Ubuntu中删除用户:

sudo userdel -r newuser

参数: -r 删除家目录[还是台湾叫法好啊]
