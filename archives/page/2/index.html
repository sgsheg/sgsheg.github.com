《我编程 我快乐》阅读笔记

我编程 我快乐[程序员职业规划之道]

  1. 序

我很懊悔.世界不会因为我没有完成工作而停滞不前,我讨厌这种感觉,这让我觉得我的存在毫无意义.

我认为从事卓越的职业是拥有意义的人生的起点.你不仅成为了一名更优秀更具有价值的员工,更成为了一个更有价值的一个.

这就是这本书的重要之处.它不是教你如何做出一个更好的小程序并体会到工作的稳定感,而是帮助你成就更卓越,更有意义的人生,工作只是这其中的一部分.

  1. 前言

如何生活的大部分时间都被工作占据,那么热爱工作就是热爱生活.

一旦满足生存的基本需求,人就会转而追求快乐.

我要告诉你,你要出色,你要赢.就像在赛跑中,如果你总想着怎么不输,那肯定赢不得比赛.

任何人都不应该时刻想着如何避免失败!!

公司雇我们,绝不是因为他们爱我们.事实上,他们以前没有爱过我们,将来也不会.

要评定自身的表现,就要看你能给你的老板创造多少商业价值.

css3实用指南阅读笔记1

css3实用指南阅读笔记

写在前面

尽可能广泛的受众提供尽可能优质的体验

仅仅因为部分人不愿意或者不能升级浏览器,却让使用新型浏览器的用户无法享受CSS3所提供的伟大技术,这是毫无道理可言的

更少的标记和图片意味着用户需要下载的内容会更少,页面加载会更快.更少的图片,脚本和flash文件也就意味着更少的HTTP请求数,这就是提高页面加载速度的最佳方法之一.

某些图形化的CSS3技术,比如渐变,能够减少HTTP请求,但可能要求浏览器执行更多的工作来渲染,这就可能导致浏览器响应缓慢,并减少可用性.

因此不要随意滥用复杂的特性,并要完整地测试实现的那些特效.

有时候成功的使用CSS3最主要的阻碍反而不是浏览器的支持度,而是来自于客户或者老板的犹疑.

常见的CSS属性的浏览器前缀:

  1. -ms- Trident Internet Explorer

  2. -moz- Mozilla Firefox

  3. -o-* Presto Opera,Opera Mobile,Opera Mini,NIntendo wii浏览器

  4. -webkit- Webkit Safari,Safari IOS,Chrome,Android浏览器

CSS性能问题

CSS性能问题

与网页接触很久了,从最初的了解信息,到后来的developer.最近的两个月内PHP写的少了,less和js倒是写的不少了,虽然服务器上将less解析为css的脚本是我写的,无奈在前端开发中又岂止当初的浮动,盒子模型.less和sass的出现,让CSS的书写更加顺手.

CSS的渲染速度决定于哪里?这个因素很大.对于php来说,smarty渲染,网络速度,页面加载速度,有很多的因素.但不管如何,知道CSS的一些好写法,未尝不可.要不然,像CSS3这样的屠龙刀,你要用它劈柴,呵呵,你还是换个吧,这么重的玄铁,还挺费手!

看完以后,总结如下:

开头第一句话,

Put CSS in the document head!!!

  1. id选择器和class选择器上,多数人觉得id比class快,但事实上二者的区别不大.可以不做考虑

  2. CSS3的绚丽特性犹如小米,才是页面渲染的最大考验.animation和@font-face,请谨慎使用.对于可以不使用CSS3属性,比如text-shadow,那就尽量不使用.

  3. 对于ul > li >a[title="sgsheg"]这样的代码,最先读到的部分是a[title="sgsheg"]

  4. 选择器的效率,从高往低来讲,依次是id > class > tag > universal.

  5. 不需要在使用id选择器时,使用ul#sgsheg,因为id是唯一的.ID selectors are unique by definition.

  6. 对于tag标签ul等,也要与class分开写.

  7. 后代选择器是性能最差的,譬如ul > li > a {},这样的使用是很糟糕的.记住,浏览器的渲染是从右往左的.这样的代码,只会更加的累赘.Why not give the a an class to render?对于这样的,为什么不给它个class,.alink { color: red}

  8. 在浏览器渲染页面时候,如果出现错误,那么它就会跳过这段代码,继续进行下面的.

  9. 在书写CSS的时候,you need think more,是否可以使用继承,继承才是代码艺术.就像jQuery那样,write less, do more.否则,我只能说”呵呵,请接受治疗!”

  10. 停止使用@import,请使用link引入.请把Javascript位于CSS之后.

  11. 在手机上,最牛叉的配置也赶不上十年前的电脑,所以开发手机网页要谨记,耍酷要看平台.

当然,原文部分在这里:

  1. Moving inline style blocks and elements from the document body to the document head improves rendering performance.

  2. Use the tag instead of CSS @import

  3. CSS 3 properties are some of the worst offenders when it comes to page performance. Especially animations and @font-face. Use them wisely.

  4. Generally speaking, if a page is slow to resize it’s likely a layout property (float, fluid width, etc.) and if it’s slow to scroll it’s likely an aesthetic property (such as shadows and gradients) that’s slowing the page down.

  5. Mobile is a thing and even the best mobile phones aren’t as powerful as the best computers from a decade ago.

Why not try a test?
