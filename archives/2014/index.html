2014第一篇工作总结

近期在做移动和后台管理系统，随着科技发展，也在不断的增长着知识。总结一些自己没有真正掌握的技术点：

  1. html

    • input->readonly属性，在后台部分，当table列表中用户没有权限去修改行内容时候，<input readonly="readonly"很好解决了这个问题。而减少了一些复杂的js控制。

    • 图片在多设备展示问题，项目中遇到一个问题。一张图片所在div既要满足于设备的宽度又要在几乎所有设备上面展示相近，那么合理的去使用流式布局是一种很正确的思路，唯一麻烦的地方是要根据设计稿来计算百分比以达到几乎完美的境地。这需要耐心与不断的测试。

    • 图片在基本html中，如果仅仅是一张图片，要记住也要给图片设置百分比宽高。

        <img src="1.jpg" width="100%" height="100%"/>

    • 满足两个div无论设备分辨率大小都要始终满足登高，特别是在移动设备上。这个时候不得不用到纯生js.

        //页面初次加载
  xxx.style.cssText

  //windo.resize
  xxx.style.cssText

      -- 期间要注意多个元素获取时候，要使用length来进行判断对象是否存在

    • 切莫自己去多写bootstrap的新css,使用版本是2.3版本，bootstrap封装了很多的好用的属性，不要为了某些特殊的效果而在bootstrap上比如span2,span12，这个时候何不多追加一个class去覆盖呢。

      // 如何去更改Input的宽度 可以使用

        input-mini

  input-small

  input-medium

  input-large

  input-xlarge

  input-xxlarge

  2. javascript

    • 减少全局变量，多用局部变量

    • 学会封装,使用iife

    • $(this), this, winow

再看提升网站性能最佳实践

  1. 第一类 [内容]

    • 尽量减少HTTP请求

      • 合并文件，如果可能，合并你的css,js文件

      • 使用css sprites合并你的图片

      • 在某些特殊页面可以考虑使用图片地图(map)，但是不推荐

      • 使用内联图像（data:URL scheme），目前多数浏览器尚未支持

    • 减少DNS查找

      • 一般情况下返回给定域名对应的IP地址会花费20到120毫秒的时间，而且在这个过程中都不会直到DNS查找完毕

      • 缓存DNS

      • 减少主机名的数量可以减少dns查找次数

《代码的未来》阅读下

  1. 作为系统描述语言的C和C++,作为商务的语言而入日中天的Java,以及在web领域中十分热门的Ruby,perl,Python,PHP等脚本语言,都属于主流语言.这些语言在成长过程中都吸收了过去一些语言的优点,例如Java借鉴了C++,Smalltalk和Lisp的优点,而Ruby则是在吸收了过去语言的基础上而独自发展的语言.

  2. 仅仅将过程进行抽象化还远远不够.几乎所有的过程都需要进行一定的输入输出操作,而不是与数据无关.在数据抽象化的延长线上,就自然而然产生了面向对象编程的概念.所谓抽象,就是抽象化数据本身.

  3. 因此面向对象和数据抽象化之间仅仅隔了薄薄的一层纸.21世纪面向对象已经是常识了.
